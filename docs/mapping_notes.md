# Заметки по маппингу OpenFOAM → FlowVision

## Модели турбулентности

| OpenFOAM | FlowVision | Примечания |
|----------|-----------|------------|
| `kOmegaSST` | SST | Прямое соответствие |
| `kEpsilon` | KES | Стандартная k-ε |
| `realizableKE` | KEFV | k-ε модификация FlowVision |
| `SpalartAllmaras` | SA | Одноуравненная модель |
| `Smagorinsky` | Sm | LES (требует соотв. сетки) |
| `kOmega` | SST | В FV нет чистой k-ω, используем SST |
| `laminar` | — | Ламинарный режим |

## Граничные условия

| OF тип | Поле | FV тип | Примечания |
|--------|------|--------|------------|
| `fixedValue` | U | Inlet/Outlet | Задана скорость |
| `fixedValue` | p | Inlet/Outlet | Задано давление |
| `fixedValue` | T | Wall | Задана температура стенки |
| `freestreamVelocity` | U | Non-reflecting | Freestream → неотражающее |
| `freestreamPressure` | p | Non-reflecting | — |
| `pressureInletOutletVelocity` | U | Inlet/Outlet | Переключаемое условие |
| `noSlip` | U | Wall | Стенка без проскальзывания |
| `slip` | U | Symmetry | Стенка с проскальзыванием |
| `zeroGradient` | любое | Free outlet | Нулевой градиент |
| `symmetry` | — | Symmetry | Плоскость симметрии |
| `symmetryPlane` | — | Symmetry | То же |
| `empty` | — | Symmetry | 2D-ограничение → симметрия |
| `cyclic` | — | Connected | Периодическое BC |

## Давление

**Ключевая особенность:** `simpleFoam` / `incompressibleFluid` в OpenFOAM
хранит **кинематическое давление** p/ρ (единицы м²/с²).

FlowVision использует **абсолютное давление** в Па.

Формула конвертации:
```
P_fv = P_ref + p_of × ρ
```

где:
- `P_ref` = 101325 Па (атмосферное, задаётся в CGeneralSettings)
- `p_of` — значение из OpenFOAM
- `ρ` — плотность вещества

Для `interFoam` / VoF давление `p_rgh` уже динамическое (Па),
конвертация проще: `P_fv = P_ref + p_rgh`.

## Температура

OpenFOAM может использовать T в К или °C в зависимости от солвера.
FlowVision всегда работает в К.

Если в OF `dimensions = [0 0 0 1 0 0 0]` (кельвин), конвертация 1:1.

## Геометрия и сетка

FlowVision **не импортирует** polyMesh напрямую. Подход:

1. Каждый патч polyMesh экспортируется в отдельный binary STL-файл
2. Полигональные грани разбиваются на треугольники (fan triangulation)
3. STL импортируется в FlowVision через «Файл → Импорт геометрии»
4. FlowVision строит собственную сетку по импортированной геометрии

**Ограничение:** параметры адаптации сетки (размер ячеек, уровни
вложенности) нужно задать вручную в FlowVision после импорта.

## Турбулентные параметры на границах

OpenFOAM задаёт k, ε/ω напрямую на BC. FlowVision вместо этого
использует интенсивность пульсаций I_t и масштаб турбулентности L_t.

Конвертер автоматически пересчитывает параметры по формулам,
идентичным в обоих пакетах (FlowVision doc.pdf Turb-BC.15–18,
OpenFOAM User Guide k-epsilon / k-omega SST):

Прямые формулы (I, L → k, ε/ω):
```
k   = 1.5 × (U × I_t)²
ε   = Cμ^0.75 × k^1.5 / L_t        (k-epsilon: KES, KEAKN, KEFV, KENL)
ω   = k^0.5 / (Cμ^0.25 × L_t)      (k-omega SST)
```

Обратные формулы (k, ε/ω → I, L), используемые конвертером:
```
I_t = sqrt(2k/3) / |U|
L_t = Cμ^0.75 × k^1.5 / ε          (k-epsilon)
L_t = k^0.5 / (Cμ^0.25 × ω)        (k-omega SST)
```

где Cμ = β* = 0.09.

При |U| = 0 используется значение по умолчанию I_t = 0.05.

Реализация: `convert_turbulence_params()` в `model_mapper.py`.
Применяется автоматически в `apply_mapping()` для BC и IC.

## Обледенение

Модуль «Обледенение» FlowVision входит в решатель «Движение
многофазной сплошной среды, теплоперенос и массообмен», однако
лицензируется отдельно (см. https://flowvision.ru/services/flowvision-cost).
Для работы модуля требуется:
- Отдельная лицензия на модуль «Обледенение»
- Настройка модели кристаллизации вручную
- Задание параметров капельной фазы

Конвертер определяет кейс icing по солверу (`multiRegion`,
`chtMultiRegionFoam`) и выбирает соответствующий шаблон,
но детальная настройка модели кристаллизации — ручная.
