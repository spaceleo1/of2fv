# Руководство пользователя OF2FV Converter

## Установка

### Требования

- Python 3.11+
- FlowVision 3.16+ (для шаблонов и проверки результатов)
- Linux (тестировалось на Arch Linux)

### Установка зависимостей

```bash
cd diplom
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Пакет `PyQt6` устанавливается автоматически из `requirements.txt`
и необходим для GUI-интерфейса.

### Настройка FlowVision

Для работы кнопки «Открыть в FlowVision» установите FlowVision
в директорию `flowvision/flowvision/` рядом с проектом, либо
задайте переменную окружения:

```bash
export FVPPP=/path/to/FlowVision/FvPPP
```

## Запуск

### GUI-интерфейс

```bash
python gui.py
```

### CLI-интерфейс

```bash
python convert.py -i <путь_к_OF_кейсу> -o <выходная_директория>
```

Параметры CLI:

| Флаг | Обязательный | Описание |
|------|:---:|----------|
| `-i`, `--input` | да | Путь к директории кейса OpenFOAM |
| `-o`, `--output` | да | Директория для выходного проекта |
| `-n`, `--name` | нет | Имя проекта FlowVision |
| `-v`, `--verbose` | нет | Подробный лог (DEBUG) |

### Верификация

```bash
python verify.py <путь_к_OF_кейсу> <путь_к_.fvinp>
```

## Работа с GUI

### Шаг 1: Загрузка кейса OpenFOAM

1. Нажмите **«Обзор...»** рядом с полем «OpenFOAM кейс»
2. Выберите директорию кейса (должна содержать `0/`, `constant/`, `system/`)
3. Нажмите **«Загрузить кейс»** на панели инструментов
4. Дождитесь заполнения таблицы предпросмотра

В таблице «Предпросмотр проекта» отобразятся:
- Тип задачи (aero / vof / icing)
- Солвер OpenFOAM и соответствующая модель турбулентности FlowVision
- Вещества с плотностью и вязкостью
- Начальные условия (скорость, давление, VOF)
- Граничные условия с маппингом OF -> FV

### Шаг 2: Настройка выходной директории

1. Укажите выходную директорию (по умолчанию создаётся рядом с кейсом)
2. Опционально задайте имя проекта

### Шаг 3: Конвертация

1. Нажмите **«Конвертировать»**
2. Прогресс отображается в панели «Лог»
3. По завершении появится диалог с количеством файлов

Результат:

```
output_dir/
  project/              ← проект FlowVision (открывать в FvPPP)
    *.fvproj, *.fvinp, *.fvctrl, ...
  geometry/             ← STL-файлы патчей
  conversion_report.txt ← текстовый отчёт
```

### Шаг 4: Верификация

1. После конвертации активируется кнопка **«Верифицировать»**
2. Нажмите её — откроется вкладка «Верификация»
3. Таблица покажет сравнение параметров OF и FV

Статусы:
- **OK** — значение совпадает
- **OK (mapped)** — значение корректно маппится
- **~** — приблизительное совпадение (напр., вещество из БД FV)
- **FAIL** — несовпадение (требует внимания)

### Шаг 5: Открытие в FlowVision

1. Нажмите **«Открыть в FlowVision»**
2. FlowVision PPP откроет сгенерированный проект
3. Импортируйте STL-геометрию из `geometry/`
4. Назначьте граничные условия на группы фасеток
5. Настройте расчётную сетку

## Поддерживаемые типы задач

| Тип | Солверы OpenFOAM | Шаблон FV | Особенности |
|-----|------------------|-----------|-------------|
| Аэродинамика | `simpleFoam`, `pimpleFoam`, `incompressibleFluid` | NACA0012_3deg | SA, SST, k-epsilon |
| VoF (гидродинамика) | `interFoam`, `incompressibleVoF` | Boat | Двухфазные задачи |

## Типичные ошибки

### «polyMesh not found»

Кейс не содержит готовой сетки. Запустите `blockMesh` в OpenFOAM
перед конвертацией, чтобы сгенерировать `constant/polyMesh/`.

### «EXCEPTION DURING OPENING A PROJECT»

FlowVision не открывает проект. Возможные причины:
- В директории `project/` есть посторонние файлы — удалите всё кроме FV-файлов
- Лицензия FlowVision недоступна — проверьте подключение к серверу лицензий
- Проект уже открыт в другом экземпляре FvPPP — закройте его

### Несовпадение значений при верификации

Некоторые параметры невозможно перенести точно:
- Вещества берутся из базы данных FlowVision (приблизительное совпадение)
- Турбулентные BC: OpenFOAM задаёт k/omega, FlowVision — интенсивность пульсаций
- Давление в OF может быть кинематическим (p/rho)

## Ограничения

- Расчётная сетка не переносится напрямую — только геометрия (STL)
- Результаты расчётов не переносятся
- Substance DB FlowVision не редактируется через XML
- Icing-кейсы требуют отдельной лицензии на модуль «Обледенение» (входит в решатель «Движение многофазной сплошной среды, теплоперенос и массообмен»)
